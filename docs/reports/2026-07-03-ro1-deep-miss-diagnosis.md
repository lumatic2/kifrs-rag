# RO1 — 잔여 miss 진단 리포트

> Generated: 2026-07-03
> Horizon: `docs/horizons/rag-optimization-resume.md`
> Milestone: RO1 (`docs/plans/2026-07-03-ro1-residual-miss-diagnosis.md`)

## 요약

M4(2026-06-27) 종료 시점 hybrid recall@20 miss 9건을 재진단했다. 전역 랭크(hybrid/hierarchical/
reranked, limit≤200)를 직접 조회한 결과 **얕은 랭킹 2건은 이미 해결돼 있었다** — 코드 변경 불필요,
관측 오류를 바로잡는 것으로 완료. 나머지 **깊은 랭킹 7건**은 후보 풀(lexical+semantic 상위 100
문단 union)에 아예 없어 재랭킹으로 해결 불가 — 3개 원인 카테고리로 분류했다.

## 1. 얕은 랭킹 2건 — 이미 해결됨 (코드 변경 없음)

| 문항 | 정답 | hybrid rank | hierarchical rank | **reranked rank** |
|---|---|---|---|---|
| Q004 | 1001-69 | 31 | 41 | **6** |
| Q041 | 1102-11 | 22 | 18 | **13** |

**발견**: `kifrs/eval/retrieval.py`의 "문항별 정답 miss" 출력은 항상 **hybrid 기준**(코드 주석
"hybrid 기준, 있으면")이라, `reranked`가 이미 두 문항 모두 top-20 안에 넣고 있는데도 콘솔 출력에는
계속 "miss"로 나타난다. `search_reranked`(candidates=50 기본값)로 직접 조회하면 Q004는 순위 6,
Q041은 순위 13 — **候補풀 확대(50→100) 실험은 필요 없었다**. 실제 결론은 "코드가 아니라 눈에 보이는
리포트가 오해를 유발했다"는 것.

- **조치**: 코드 변경 없음(이미 정상 동작). 다만 이 horizon 재개 계기가 된 오해(hybrid 표에서만
  본 miss 9건을 "전부 개선 필요"로 읽은 것)를 바로잡기 위해 이 리포트에 기록.
- **후속 제안(경량)**: `kifrs/eval/retrieval.py`의 miss 출력을 `--retrievers`로 넘긴 여러
  retriever 각각에 대해 따로 보여주면 이런 오독을 막을 수 있다 — RO2/RO3 후보로 남김(작은 tooling
  개선, 별도 changeset 판단은 사용자와 논의).

## 2. 깊은 랭킹 7건 — 후보 풀 밖 (재랭킹으로 해결 불가)

전부 hybrid/hierarchical/reranked 세 retriever 모두에서 rank > 70 (100-union 후보 풀 밖 또는
경계) — 재랭킹이 아니라 **1차 검색(lexical+semantic) 자체가 후보로 못 끌어옴**.

| 문항 | 정답 | 질문-문단 키워드 overlap | hybrid rank |
|---|---|---|---|
| Q001 | 1115-27 | 없음(사실상) | >89 |
| Q040 | 1109-4.1.4 | **없음** | >84 |
| Q006 | 1115-51 | 제품, 환불 | >97 |
| Q008 | 1109-2.1 | 1109, 금융상품, 리스부채, 리스이용자, 인식한 | >83 |
| Q029 | 1116-45 | 리스, 리스부채 | >81 |
| Q039 | 1037-14 | 의무, 충당부채 | >84 |
| Q048 | 1036-18 | 회수가능액 | >70 |

### 카테고리 A — 순수 어휘 부재형 (질문·조문 어휘 자체가 안 겹침)

**Q001, Q040.** 질문은 구체 사례형 서술("소프트웨어 라이선스와 설치용역 패키지 판매", "신용연계
채권의 분류")이고 조문은 추상 정의·규칙 문장이라 표층 어휘가 거의 안 겹친다. Q040(1109-4.1.4)은
오늘 세션의 RGA1(citation grounding) 작업에서 **완전히 독립적으로 동일한 문단**을 keyword
overlap=0/cosine 유사도로 확인한 적이 있다 — 같은 문단이 두 번 다른 각도(인용 검증 vs 검색 recall)
에서 "표층 어휘 매칭 실패"로 나타난 셈. `CLAUDE.md` 한계 #1과 정확히 같은 패턴.

- **해법 후보**: M3b식 term bridge(시험표현→본문표현) 확장. 다만 M3b는 "반복되는 일반 패턴"에
  효과적이었고, 이 두 건은 사례별 특이 표현이라 일반화 가치가 낮다 — 사례 하나하나 dict에 추가하는
  건 확장성이 떨어진다(수확체감).

### 카테고리 B — 일반어 매칭, 판별력 부족 (겹치지만 흔한 용어라 특정 안 됨)

**Q006, Q008, Q029.** 질문과 조문이 키워드를 공유하지만("리스", "리스부채", "금융상품", "환불" 등)
그 표준 안에서 반복적으로 나오는 흔한 용어라 BM25/임베딩 둘 다 특정 문단을 골라내지 못한다. 예:
1116(리스)에는 "리스부채"가 들어간 문단이 수십 개, 1109(금융상품) scope 조항(2.1)도 "금융상품"
자체가 표준 전체에서 흔한 단어라 IDF 가중치가 낮다.

- **해법 후보**: 문단 메타데이터(조·항 제목, 섹션명)를 임베딩/BM25 신호에 추가 가중 — 이미 M4가
  "섹션 centroid"로 유사한 시도를 했으나(hierarchical) 이 3건은 hierarchical에서도 회복 안 됨
  (Q006 rank>139, Q008 rank>103, Q029 rank>101 — 오히려 hybrid보다 나쁨). 섹션 신호로도 안 잡히는
  걸 보면 더 정밀한 신호(문단 고유 키워드 TF-IDF 가중 등)가 필요 — 비용 대비 효과 불확실.

### 카테고리 C — 크로스 개념/표준 쏠림 (질문이 여러 개념을 걸침)

**Q039, Q048.** 질문이 두 개념을 걸친다: Q039는 "리스 종료 원상복구 의무"(1116) + "충당부채
인식요건"(1037) — 질문 임베딩이 리스 쪽 어휘에 쏠려 1037-14가 밀린다(M4가 이미 "섹션 centroid
전역순위 674"로 진단한 바로 그 문항). Q048은 "회수가능액 정의"(1036-18) + "손상차손 인식"(1036-59,
다른 must_cite) — 질문 임베딩이 한쪽으로 쏠리며 다른 쪽 문단이 희석된다.

- **해법 후보**: 질문을 하위 개념별로 분해해 각각 별도 쿼리로 검색(멀티 쿼리 검색) 후 결과 병합.
  M1~M4 어느 milestone도 시도하지 않은 새 접근 — 카테고리 A/B보다 일반화 가능성이 높아 보인다
  (cross-standard 질문은 goldset에 여러 건 있을 가능성).

## RO2 착수 판단

카테고리별 비용 대비 가치:
- **A(2건)**: 사례별 term bridge — 확장성 낮음, 착수 가치 낮음(M4-2와 같은 "가치 낮음" 판단).
- **B(3건)**: 메타데이터 가중 — hierarchical도 못 잡은 걸 보면 난이도 높고 효과 불확실.
- **C(2건)**: 멀티 쿼리 분해 — 새 접근, 이 goldset 밖에도 일반화될 가능성 → **RO2 후보 중 가장
  가치 있어 보임**.

**권고**: RO2는 카테고리 C(멀티 쿼리 분해)만 좁게 시도. A/B는 착수 보류(신호 낮음, 필요 시 재검토).
최종 착수 여부는 horizon doc 갱신 시 사용자와 확정.
