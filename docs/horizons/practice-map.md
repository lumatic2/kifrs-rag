# 업무 지도 (Practice Map) Horizon

> Created: 2026-07-04
> ROADMAP goal id: `practice-map`
> Status: active
> Objective: `docs/OBJECTIVE.md`

## Why now

Objective가 2026-07-04에 프로덕트 지향으로 재정의됐다 — "회계사 업무를 AI로 어디까지 자동화할
수 있는가"에 실증으로 답하고 법인에 소개 가능한 프로덕트를 만드는 것. 그런데 지금까지의 도메인
선택(1109 → 1116 후보)은 **기준서 구조 기준**(결정론적인 것부터)이었지, 실제 회계사가 시간을
쓰는 업무의 빈도·가치 기준이 아니었다. "어디까지 가능한가"에 답하려면 먼저 "회계사 업무 전체"가
지도로 존재해야 한다 — 지도가 없으면 완료율 축은 좌표 없는 점 하나다.

관찰된 시작 데이터(2026-07-04, 사용자): 주변 회계사들은 AI를 적극 활용하지 않으며, 회계법인
사내 AI는 주로 리서치·자료 정리 용도다. 이 관찰 자체가 지도의 첫 항목이다 — "리서치·자료정리 =
이미 AI가 닿은 영역, 그 너머가 공백".

## Goal

회계사 실무 업무 taxonomy(감사·재무제표 작성·자문·세무 등 → 세부 task 단위)를 작성하고, 각
업무에 자동화 가능성 판정("가능 / 조건부 가능 / 불가 / 미실험")과 근거를 매핑한다. 기존 자산
(1109 엔진, 검색 인프라, /accounting)의 지도상 위치를 표기하고, **다음 자동화 대상을 기준서
구조가 아니라 실무 가치 기준으로 선정**할 수 있는 상태를 만든다.

Objective 축 1(업무 지도 커버리지)의 측정을 이 horizon이 시작한다.

## Milestone candidates (2~5, horizon-run continuation용)

1. **PM1 — 회계사 업무 taxonomy 초안** (first, this planning round)
   공개 자료 리서치(한공회·법인 공시자료·채용 JD·실무 가이드 등) 기반으로 업무 분해 문서 작성.
   대분류(감사/재무제표/자문/세무/기타) → 세부 task, 각 task에 빈도·판단강도·입출력 형태 메타.
   기존 자산(1109 엔진, Phase 3/4 시나리오, /accounting 리서치 능력)의 위치 표기 포함.
   learning 갈래 성격(리서치·분석 산출물).
2. **PM2 — 현업 검증 (회계사 인터뷰/피드백)** (candidate, 사용자 액션 필요)
   주변 회계사에게 PM1 taxonomy를 보여주고 실제 업무 비중·pain point·AI 활용 현황을 검증/보정.
   **사용자 소유 액션**(외부 사람 필요) — 착수 시점은 사용자가 정한다. PM1 산출물이 인터뷰
   재료가 되므로 PM1 뒤.
3. **PM3 — 자동화 가능성 매핑 + 다음 대상 선정** (candidate)
   taxonomy의 각 task에 "가능/조건부/불가/미실험" 판정 + 근거(기존 실험 결과 또는 판단 사유)를
   붙이고, 커버리지 축 첫 측정값을 기록. 다음 자동화 대상(도메인/업무)을 실무 가치 기준으로
   1~2개 선정해 후속 horizon(자동화 확장)의 입력으로 넘긴다.

## Close criteria

PM1 + PM3가 닫히면(taxonomy 존재 + 가능성 매핑 + 다음 대상 선정) 이 horizon의 핵심이 닫힌다.
PM2(현업 검증)는 사용자 일정에 종속 — PM2 없이도 horizon을 조건부로 닫을 수 있되, 그 경우
지도의 "미검증(공개자료 기반)" 상태를 명시하고 PM2를 후속 horizon으로 이월한다.

## 후속 horizon 후보 (이 horizon이 끝나면)

- **자동화 확장** — PM3가 고른 업무의 엔진화. WA2(1116 이식)·WA2b(1109 잔여 4건)가 이 안으로
  흡수될 수 있다 (paused `workflow-automation` horizon과 통합 검토).
- **프로덕트 패키징** — 설치 가능한 도구킷 정비 + 데모 자료 + 회계사 실사용 피드백(성공 모습
  중간 관문 2·3).
