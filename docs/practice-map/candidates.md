# 다음 자동화 대상 후보 심층 분석 (PM3 Step 2)

> Created: 2026-07-04 · Milestone: PM3 · 입력: `taxonomy.md` PM3 판정 (미실험-유망 + 조건부 실증)
> 제약: 로컬에서 AI+웹검색으로 검증 가능해야 함 (현업 접촉 보류 — 2026-07-04 사용자 결정)

후보 5개 = 판정 "미실험-유망" 중 로컬 검증성 상위 4개(B5·A5·E2·D3) + 기존 실증의 직접 확장 1개
(B3-확장). D2(가치평가)·B4(F/S 본문)는 각각 D3·B5와 겹쳐 대표 후보에 흡수.

## 후보 1 — B5: 주석(공시) 초안 생성 ★ 추천 1순위

| 항목 | 내용 |
|---|---|
| 필요 입력 | ① 기준서별 공시 요구사항 문단 — **이미 우리 DB에 있음**(1007 현금흐름표, 1107 금융상품 공시, 1116 리스 공시 등) ② 회사 수치(시산표/F-S) — 자작 fixture 또는 DART 공개 F/S ③ 검증 골드셋 — **DART 공시 주석 = 무료·무제한 공개 데이터** |
| 자산 재사용 | 검색 인프라(공시 요구사항 검색 = A8 실증 경로 그대로), 1109 엔진 패턴(구조화 입력→결정론 산출→회귀 fixture), goldset 방법론(M1 하네스) |
| 검증 방법 | 도메인 1개(예: 1116 리스 주석)의 요구사항 체크리스트를 엔진화 → DART 실제 상장사 리스 주석과 대사 → "요구항목 커버리지 %" 측정 — 완료율 축과 같은 방식의 신규 측정치 |
| 예상 실패 모드 | 주석은 회사 특수 서술 많음(초안까지만 — Objective와 정합) · 요구사항 문단→체크항목 변환의 정확도 · DART 주석 파싱 품질 |
| Objective 정합 | 성공 모습에 "주석 초안" 명시. 법인 AI 공백 지대(판단 본질 task). **신규 task 개척** — 커버리지 축을 3/33으로 확장 |

## 후보 2 — B3-확장: 1116 리스 결정 엔진 이식 ★ 추천 2순위

| 항목 | 내용 |
|---|---|
| 필요 입력 | **전부 이미 있음** — 1116 WORKFLOW.md + 시나리오 fixture 10개(`data/scenarios/1116_lease/`) |
| 자산 재사용 | 1109 엔진 9-모듈 패턴 통째로(schema/판단/classify/entry/amortization/review_memo/fixtures/runner), grounding 레이어(RGA1), 회귀 하네스 |
| 검증 방법 | WA1과 동일 — 10개 fixture 회귀, 완료율 2번째 도메인 측정 ("엔진 패턴이 이식되는가" = paused workflow-automation horizon의 닫는 기준 (a)) |
| 예상 실패 모드 | 1116은 재평가(reassessment)·변경 트리거가 1109보다 판단적 — NeedsHumanReview 비중이 더 클 수 있음(그것도 유효한 측정) |
| Objective 정합 | 완료율 축을 벡터(2-도메인)로 만듦. **기존 실증 심화** — 리스크 최소, 신규성은 낮음 |

## 후보 3 — A5: 분석적 절차 (재무비율·추세 분석)

| 항목 | 내용 |
|---|---|
| 필요 입력 | DART Open API(무료) 공개 F/S — 로컬 확보 완벽 |
| 자산 재사용 | 거의 없음(신규 계산 모듈) — 기준서 DB와 무관 |
| 검증 방법 | 계산 자체는 결정론적(검증 쉬움), 단 "이상 징후 해석"은 판단 영역 |
| 예상 실패 모드 | 계산기 수준을 넘는 가치(패턴 해석)를 내려면 결국 감사 맥락(내부 기대치) 필요 — 이 레포 강점과 접점 약함 |

## 후보 4 — E2: 사업비 정산 검증

| 항목 | 내용 |
|---|---|
| 필요 입력 | 정산 지침(공공 공개) + 증빙 fixture(자작 필요 — 제작 비용 있음) |
| 자산 재사용 | 엔진 패턴은 재사용 가능, 기준서 DB는 무관 |
| 검증 방법 | 자작 fixture 회귀 — 가능하나 골드셋 신뢰성이 자작에 갇힘 |
| 예상 실패 모드 | K-IFRS 레포 정체성과 거리(보조금 정산은 별도 도메인), 시장도 니치 |

## 후보 5 — D3: 비상장 주식평가 (상증법 보충적 평가)

| 항목 | 내용 |
|---|---|
| 필요 입력 | 재무자료(fixture 가능) + 상증법 산식 — **세법: tax-agent 경계 침범** |
| 자산 재사용 | 산식은 결정론적이라 엔진 패턴 적합, 단 근거 DB가 세법(이 레포 밖) |
| 검증 방법 | 법정 산식이라 검증 쉬움 |
| 예상 실패 모드 | 경계 위반 — Objective 결정 ⑤(세무는 tax-agent)와 충돌. **이 레포가 아니라 tax-agent 후보로 이관 권고** |

## 추천 (최종 선택은 사용자 — 다음 horizon 논의 입력)

**1순위 B5(주석 초안) + 2순위 B3-확장(1116 엔진)** — 상호보완적:
- B5는 **새 task 개척**(커버리지 축 확장, 법인 AI 공백 직격, DART로 로컬 검증 완벽) — 리스크는
  신규 파이프라인(주석 파싱·체크리스트 변환).
- B3-확장은 **기존 실증 심화**(완료율 축 2-도메인화, fixture 완비로 착수 비용 최소) — 리스크
  최소, 프로덕트 스토리("여러 도메인에서 되는가")에 필요.
- 순서 제안: 둘 다 다음 "자동화 확장" horizon에 담되, **B3-확장(1116)을 먼저**(패턴 이식 검증,
  1~2주 grain) → **B5(주석)를 두 번째**(신규 개척). 이유: B5는 DART 파싱 등 신규 인프라가 필요해
  실패 모드가 많고, 1116은 지금 바로 회귀 가능한 fixture가 있어 확실한 전진을 먼저 확보.
- A5·E2는 보류(접점 약함/니치), D3은 tax-agent로 이관 권고.
