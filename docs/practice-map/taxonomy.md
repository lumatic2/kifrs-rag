## 메타 필드 정의

- **빈도**: 상시 / 시즌반복(감사시즌 등 기간 집중) / 월·분기 / 연1회 / 건별(프로젝트) / 수시
- **판단강도**: 기계적(규칙 따라가면 됨) / 중(패턴 있으나 예외 판단) / 고(전문가 판단이 본질)
- **입출력**: 입력 → 출력 형태
- **현 AI활용**: 높음(업계 도구 존재·보편) / 부분(대형 법인 도구 있으나 제한적) / 낮음(사실상 수작업) / 불가성격(대면·현장·책임)

## 관찰: 현 AI 지형의 요약 (출처 15~20)

- 대형 법인 AI는 **A8(리서치)·A6(문서 대량 처리)·조서 표준화**에 집중 — 사용자 관찰("리서치·자료
  정리 수준")과 일치한다.
- **판단이 본질인 task(B3 회계처리 판단, A10 감사보고서, B5 주석)는 공백** — 법인 도구도 여기는
  "사람 중심" 명시(EY 보도자료). 이 공백이 Objective가 노리는 차별점과 정확히 겹친다.
- 기계적 task(B1 기장, C2·C3 신고)는 이미 SW로 포화 — 프로덕트 기회 아님.
