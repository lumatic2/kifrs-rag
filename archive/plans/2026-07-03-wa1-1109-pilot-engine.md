## Scope boundary

거래 입력(구조화) → SPPI/사업모형 분류 → 최초인식 분개 → **후속측정(상각표) 포함** → 검토메모까지
전 과정을 실행 가능한 코드로 만들고, 기존 10개 `data/scenarios/1109_classification/scenario_*`
를 회귀 fixture로 재현해 완료율을 측정한다 (2026-07-03 결정 — 후속측정 포함으로 범위 확정).

**중요 발견 (계획 단계, 2026-07-03)**: 기존 시나리오의 유효이자율(EIR)은 수치해석으로 역산하지
않고 **가정값으로 단순화**돼 있다(`scenario_01/entries.md`: "본 시나리오에서는 유효이자율 8% 가정
— 정확한 계산은 별도 검증"). FVOCI/FVPL의 기말 공정가치도 **외부에서 주어진 자료값**이다
("자료의 공정가치는 가정. 실무는 거래소 호가 또는 평가기관 데이터"). 즉 엔진은 EIR/기말FV를
**입력값으로 받아 결정론적 산술**만 하면 되고, IRR류 수치해석 루틴은 이번 범위에 불필요 —
당초 예상보다 구현 난이도가 낮다.

Out of scope (이번 milestone에서 다루지 않음, 이유):
- **EIR 자동 역산** — 위 발견대로 시나리오 자체가 가정값을 쓰므로 불필요. 필요해지면 별도 후속.
- **정교한 ECL 3-stage 손상 모델** — 시나리오에 명시된 손상 신호가 있으면 단순 규칙으로만 처리,
  없으면 스킵. 신용손상 모델링 자체가 별도 도메인 — 신호 있으면 WA2 후보로 승격 검토.
- **다른 도메인(1116/1115/1113/1019) 포팅** — WA2 candidate, 이 milestone의 완료율 결과를 보고
  결정.

## 결정 로그

- **범위: 후속측정(상각표) 포함** — 사용자 선택. 최초인식만으로는 기존 시나리오의 상당 부분(상각표·
  OCI 누적)을 검증하지 못해 완료율 측정이 부실해짐.
- **EIR 역산 불필요 (계획 단계 발견)** — 기존 시나리오가 가정값을 쓰므로, 엔진도 입력값으로 받는다.
  사용자에게 재확인 불필요 — 시나리오 자체의 기존 관행을 그대로 따르는 선택이라 새 트레이드오프가
  아님.
- **ECL/손상 단순 규칙 처리 (계획 단계 기본값)** — 정교한 모델 없이 시나리오 명시 신호만 반영.
  실행 중 손상 시나리오가 예상보다 복잡하면(예: 여러 stage 전환 로직 필요) 중단하고 사용자에게
  보고 — 새 위험/스코프 변경이므로 무중단 진행 예외.
- **fixture 인코딩은 수동 1회성** — `transaction.md` 자동 파싱은 이번 범위 밖(자연어 표 파싱은
  별도 문제). 향후 실제 신규 거래 입력은 Step 1 스키마로 직접 작성하는 것이 목표 UX.
- 이 외 예상되는 사용자 소유 결정 없음 — 위 항목들로 소진됨.
